## v2.1.0 — 修复"AI 跑得短、跳环节"问题 ⚠️ 重要修复

### 🐛 问题诊断

v2.0 重构时为了"简洁可读"，把 v1 那些"啰嗦但有强制力"的指令大量精简，
导致主流程从"执行脚本"退化成了"建议清单"。具体表现：

| # | 根本原因 | 表现 |
|---|---|---|
| 1 | §3.8 / §3.9 / §3.11 / §3.14 用「模板同上」省略 | 模型遇到"模板同上"经常**跳过整轮**或敷衍带过 |
| 2 | 字数是"目标"而非"硬下限"（"150–300 词"） | AI 总是走下限 |
| 3 | 自由辩论单次 ≤50 词、无最少次数下限 | 经常打 5–10 次就草草结束（应 ≥20 次） |
| 4 | 无"完成确认"机制 | AI 觉得"差不多"就提前进下一阶段 |
| 5 | 阶段 B 赛前规划只在 round_structure.md 写散文，无 prompt | 完全被跳过 |
| 6 | 阶段 H 赛后清理 1 段 prompt 要产 7 份稿件 | 7 份常被合并成 1 段 |
| 7 | 语气"请..."、"可以..."、"建议..." | 被理解为可选 |
| 8 | 无全场进度看板 | 编排器自己也不知道跳了什么 |

### 🆕 v2.1 修复手段

1. **`prompts/round_prompts.md` 全面重写（v2.0 字符数 9745 → v2.1 字符数 ~30000+）**
   - 消灭所有"模板同上"，§4.1–§4.14 每轮都是**完整可独立执行**的 prompt；
   - 新增 **§3B（阶段 B 赛前规划）** 5 个独立 prompt：论点架构 / 资料收集 / 攻防表 /
     头脑风暴 / 一辩稿；
   - 新增 **§6（阶段 H 赛后清理）** 6 个独立 prompt：头脑风暴 / 一辩终稿 / 二三四辩稿 ×3 /
     质询稿 ×2 / 攻防表 / 自检；明确写**"不准合并"**；
   - 新增 **§8 全场进度看板**：26 个子步骤的勾选表，编排器必须打印在对话里。

2. **字数从"目标"→"硬下限"**
   - 每轮 prompt 头部写明 `字数硬下限 ≥ XXX 字`；
   - 低于下限 → 视为本轮失败 → 编排器必须要求重做；
   - 完整一场预期输出从 6000–9000 字升级到 **≥ 12000 字**。

3. **"完成契约"机制**
   - 每轮 prompt 强制要求结尾输出 `【完成】轮次=Rxx 字数=N ...` 一行；
   - 编排器**只有读到完成契约**才允许进入下一轮；
   - 关键字段不达标 → 重发 prompt 写明"上次 X 字段不达标，请重写"。

4. **语气全面强化**
   - "请按..." → "**必须**完成以下..."
   - "可以..." → "**禁止**..."
   - 给每轮加 **失败判定条件**（如"少于 X 字一律视为本轮失败"）。

5. **关键轮次的硬约束追加**
   - **R2 / R4 首质**：≥ 4 个完整 Q-A 回合（不是只有 1-2 问）；
   - **R7 / R9 三辩质询**：≥ 3 个完整回合 + 必须小结；
   - **R10 / R11 自证**：≥ 5 段自证 + 打岔 ≤ 3 次（精确）；
   - **R12 自由辩**：≥ 20 次发言、单次 80–180 字、互相点名 100%、金句 ≥ 3；
   - **R13 / R14 总结**：5–6 节齐全 + ≥ 700 字；
   - **评委点评**：6 节齐全 + ≥ 800 字 + 必须二选一胜负。

6. **SKILL.md §6 顶层执行流**
   - 在 §6 顶部加"关键纪律"强提示框；
   - 阶段 B 从 1 行展开为 5 个子步骤的明确分发；
   - 阶段 H 从 1 行展开为 6 个子步骤的明确分发；
   - §7 长度约束从"目标"全面改写为"硬下限对照表"；
   - 新增 §7.1「完成契约」说明。

### ⚖️ 行为兼容性

- v2.0 → v2.1 **没有破坏性变化**：旧的文件名、目录结构、外部 skill 集成全部保留；
- 任何 v2.0 跑过的辩论数据都可以无缝继续用 v2.1；
- v2.1 只是让 prompt **更长、更硬、更可执行**，没有删任何旧能力。

### 📈 字数预期对照

| 阶段 | v2.0 实际 | v2.1 下限 |
|---|---|---|
| 完整 14 轮辩论 | ~3000–4000 字（跳轮） | **≥ 8000 字**（不可跳） |
| 赛前规划 4 份文件 | 经常没产出 | **≥ 1600 字** |
| 赛后清理 7 份文件 | 常被合并成 1 段 | **≥ 3400 字（7 份独立）** |
| 评委 + Synthesis | ~300 字 | **≥ 1200 字** |
| **总计** | ~4000 字 | **≥ 14000 字** |

### 🎯 使用建议

如果重跑 v2.1 后仍发现 AI 跳环节，按以下顺序检查：

1. 编排器有没有把 `prompts/round_prompts.md §8` 的进度看板打印到对话里？
2. 每一轮的子代理回复有没有 `【完成】` 契约行？
3. 字数有没有达到对应 §X.Y 的硬下限？

如果三项都是"否"，编排器违反了 v2.1 纪律，需要重发对应 prompt 强制执行。

## v2.2.0 — 时间约束 + 对话优先 ⚠️ 重要修复

### 🐛 问题诊断

用户反馈："v2.1 字数下限太低，且 AI 倾向于'写文件'而不是'演辩论'"。

诊断：
1. **字数下限过低**：v2.1 一辩 3'30" 写"≥ 600 字"，按 280 字/分钟 真实语速换算，
   实际只够 2'10"，AI 自然往下限走 → 输出短；
2. **稿件落盘喧宾夺主**：v2.0/v2.1 把"落盘 7 份文件"放在赛后清理的核心位置，
   AI 倾向于跳过对话、直接说"已生成 XX.md" → 辩论过程消失；
3. **赛前规划全在文件**：四位辩手赛前互不可见地"写"备赛资料，对话里几乎看不到讨论。

### 🆕 v2.2 修复手段

#### 1. 时间约束 + 280 字/分钟基准语速

| 时长 | v2.1 字数下限 | **v2.2 字数下限**（× 280 字/分钟）|
|---|---|---|
| 一辩申论 3'30" | ≥ 600 字 | **≥ 980 字** |
| 二辩申论 2'30" | ≥ 450 字 | **≥ 700 字** |
| 三辩质询 1'30" | ≥ 350 字 | **≥ 420 字** |
| 自证 约 3 分钟 | ≥ 500 字 | **≥ 800 字**（6 段 × 140 字）|
| 自由辩 4 分钟 | ≥ 20 次 × 80 字 | **≥ 24 次 × 140 字** |
| 总结陈词 3'30" | ≥ 700 字 | **≥ 980 字** |
| 评委点评 | ≥ 800 字 | **≥ 980 字** |

每个 prompt 顶部都明确写"按 280 字/分钟换算，时长 X，下限 Y 字"。

#### 2. 对话呈现规范（新增 §0.5）

强制所有辩论环节**在对话里完整演出**，包含：

- **标题行格式**：`**正方一辩 · 胡渐彪**（3'30"·开始计时）`
- **动作括号**：`（沉稳起立，目光扫向反方）`、`（拍桌）`、`（转向评委）`
- **真实辩场用语**：对方辩友、各位评委、主席、谢谢主席
- **质询/自由辩**呈现多人 Q-A：

  ```
  **反方四辩 · 马薇薇**：对方一辩，您把"幸福"定义为...
  **正方一辩 · 胡渐彪**：（沉吟两秒）我方的定义...
  **反方四辩 · 马薇薇**：（打断）请直接回答，是或不是？
  ```

#### 3. 赛前讨论也在对话里演

v2.1 阶段 B 的 5 个子步骤是"编排器写文件"。
v2.2 改为**四位辩手在对话里互相讨论**，编排器**最后**从对话提炼成 `备赛资料.md`：

- §3B.1 论点架构 → 四人对话头脑风暴（≥ 1200 字 / 5 分钟）
- §3B.2 资料收集 → 四人对话汇报检索结果（≥ 1000 字 / 4 分钟）
- §3B.3 攻防表 → 四人对话演练"对方会怎么打我们"（≥ 1500 字 / 6 分钟）
- §3B.4 二次修改 → 四人对话自检（≥ 800 字 / 3 分钟）
- §3B.5 一辩稿 → 一辩在对话里完整彩排 3'30"（≥ 980 字）

#### 4. 赛后清理也在对话里演

v2.1 把"赛后 7 份稿件"作为产物清单。
v2.2 强制每份稿件都先**在对话里彩排一遍**，编排器**最后**从彩排对话提炼成文件：

- §6.1 头脑风暴 → 四人对话复盘（≥ 1400 字 / 5 分钟）
- §6.2 一辩终稿 → 一辩在对话里彩排修订版（≥ 980 字）
- §6.3 ×3 → 二辩/三辩/四辩**分别**在对话里彩排
- §6.4 ×2 → 三质二、四质一各自彩排
- §6.5 攻防表 → 四人对话**逐行过**老攻防 + 新增

#### 5. 稿件 = 对话的派生物

每个对话环节末尾，由编排器输出一段提炼：

```
> **【编排器·提炼】** 一辩稿定稿，已落盘到 {side}-一辩稿.md（{N} 字）
```

**没有对话就没有可提炼的素材** → AI 不能再跳过对话直接写文件。

#### 6. 重做触发条件汇总（§9 新增）

新增 9 类典型"必须重做"的情境，最关键的是：

- ❌ 用"已生成 X.md"等元说明代替对话 → 重发，"违反对话呈现规范 §0.5"
- ❌ 赛后 7 份稿件合并成 1 段 → 重发对应 §6.X，"必须 N 份独立文件 + N 次彩排"

### 📈 字数预期对照

| | v2.0 | v2.1 | **v2.2** |
|---|---|---|---|
| 完整一场预期 | ~4000 字（跳轮） | ≥ 14000 字 | **≥ 18000 字** |
| 主战 14 轮 | ~3000 字 | ≥ 8000 字 | **≥ 11000 字** |
| 赛前 5 子步骤 | 散文/常漏 | ≥ 1600 字 | **≥ 5500 字（对话）** |
| 赛后 7 份稿件 | 常合并 | ≥ 3400 字 | **≥ 7000 字（对话彩排）** |
| 评委 + Synthesis | ~300 字 | ≥ 1200 字 | **≥ 1540 字** |

### 📐 完整一场预期总时长

**≈ 60-70 分钟"真实演讲"**（按 280 字/分钟换算）：
- 赛前对话：约 21 分钟
- 主战 14 轮：约 24 分钟
- 赛后对话：约 18 分钟
- 评委 + Synthesis：约 5.5 分钟

### ⚖️ 行为兼容性

- v2.1 → v2.2 **没有破坏性变化**：文件名、目录结构、外部 skill 集成全部保留；
- 旧的 `.learnings/` 记录可以无缝继续用；
- v2.2 只是把 prompt **改成时间约束 + 对话优先**，没有删任何旧能力。

### 🎯 使用建议

如果重跑 v2.2 后仍发现 AI 跳环节或跑得短，按以下顺序检查：

1. 编排器有没有在对话里打印 `prompts/round_prompts.md §8` 进度看板？
2. 每轮发言有没有用 §0.5 的标题行 + 动作括号格式？
3. 每轮收到的回复有没有 `【完成】` 契约行？
4. 字数有没有达到时长换算下限（按 280 字/分钟）？
5. 有没有"已生成 XX.md"代替对话呈现的情况？

如果出现任一"否"，编排器违反 v2.2 纪律，需要立刻重发对应 prompt 强制执行。
