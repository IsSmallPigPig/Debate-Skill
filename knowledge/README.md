# Knowledge Base · 华语辩论知识库

> **设计原则**：
> - 这里**不是 prompt**，而是 **prompt 的弹药库**——辩手 / 编排器 / 评委
>   在思考"这一招该怎么打"时来这里查；
> - 所有结论都有**真实出处**（具体辩手 / 具体赛事 / 具体辩题），不凭空捏造；
> - 与 `rules/script_format.md`（格式权威）分工：本目录管**怎么想、怎么打**，
>   script_format.md 管**怎么写、怎么排版**。

## 🧠 知识库与 Skill 其它部分的关系

```text
                          knowledge/  ←── 辩论"思想"
                              │
                              │ 引用
                              ▼
        prompts/round_prompts.md  ←── 辩论"动作"（每轮做什么）
                              │
                              │ 遵守
                              ▼
        rules/script_format.md    ←── 辩论"语言"（每句怎么写）
                              │
                              │ 落盘
                              ▼
        templates/                ←── 辩论"产物"（输出长什么样）
```

辩手子代理在执行任何一轮 prompt 时，**遇到需要思考"怎么打才能赢"的时候**，
应当主动 `read_file` 加载 `knowledge/` 对应文件作为参考。

## ⚙️ 使用契约

- 引用本知识库的任一论断时，**必须**标注 `[来源]` 以便核查；
- 知识库中的"招式"是模板，**不是死命令** —— 辩手应根据具体辩题灵活组合；
- 不允许把知识库的整段照搬进辩稿——必须**消化、转译**为符合 persona 语气的话。
