# Round Prompts · 提示模板库（v2.3 真实辩稿版）

> **四条铁律（务必遵守）：**
> 1. **时间约束**：每一轮 prompt 顶部写明真实演讲计时，按 **280 字/分钟** 换算等价字数下限。
>    少于下限 = 没演够时间 = 本轮失败必须重做。
> 2. **对话呈现**：所有辩论环节必须在对话里完整演出。**严禁**只写"已落盘 XX.md"就交差。
>    稿件文件是辩论**结束后**由编排器从对话提炼的派生产物。
> 3. **真实辩稿格式（v2.3 新增）**：所有辩稿/发言**严禁**带（动作/表情/语气）等小说式标记，
>    必须按 `rules/script_format.md` 的规范来写——开宗明义、综上所述、第一/第二/第三 等
>    真实辩场套语必须用上；所有产物必须可直接复制到 Word 打印。
> 4. **完成契约**：每一轮发言结束时必须输出 `【完成】` 契约行。

> **占位符约定：**
> - `{topic}`            辩题原文（每轮必须重申一遍）
> - `{persona}`          辩手中文名（如「詹青云」）
> - `{persona_skill}`    辩手 Skill 名（如 `zhan-qingyun-perspective`）
> - `{position}`         辩位（如「正方一辩」）
> - `{side}`             正方 / 反方
> - `{user_side}`        用户站的那一方
> - `{R<n>_response}`    第 n 轮的具体发言原文

> **时间↔字数换算表（基准语速 280 字/分钟）：**
>
> | 时长 | 等价字数下限 | 用途 |
> |---|---|---|
> | 30" | ≥ 140 字 | 单次自证段 / 单次自由辩发言 |
> | 1'00" | ≥ 280 字 | 短发言段 |
> | 1'30" | ≥ 420 字 | 三辩质询全场 |
> | 2'00" | ≥ 560 字 | — |
> | 2'30" | ≥ 700 字 | 二辩申论 / 首质 |
> | 3'00" | ≥ 840 字 | — |
> | 3'30" | ≥ 980 字 | 一辩申论 / 总结陈词 |
> | 4'00" | ≥ 1120 字 | — |

## 0.5 现场对话呈现规范（★ v2.3 修订）

> **核心**：呈现要像真实辩论赛**逐字记录**，**不是话剧本**。
> 详细格式见 `rules/script_format.md` §8。

### 0.5.1 申论/总结陈词的标题与正文格式

```markdown
## 第 N 轮 · {环节名}｜{辩位} · {辩手}
> 时长 X'X" · 计时方式：—

{完整辩稿正文——按 rules/script_format.md §2 或 §4 的章节结构展开。
通篇不带任何动作/表情/语气括号。直接用辩场套语：
"谢谢主席，各位评委、对方辩友、各位同学，大家好。
今天我方的立场是……"
}

【完成】轮次=RN 时长=X'X" 字数=N 含定义判准=是 含3论点=是 含证据=N
```

### 0.5.2 质询/对话轮的呈现格式

```markdown
## 第 N 轮 · {环节名}｜{质询方} → {被质询方}
> 时长 X'X" · 单边计时

**{质询方辩位} · {辩手}**：{Q1，含具体引用对方原话}

**{被质询方辩位} · {辩手}**：{A1，必须实质回答}

**{质询方辩位} · {辩手}**：{Q2}

**{被质询方辩位} · {辩手}**：{A2}

（继续，至少 N 个完整 Q-A 回合）

**{质询方辩位}（小结）**：{1 句话定性}

【完成】轮次=RN 时长=X'X" 总字数=N 完整回合=N 已钉定义=是 已做推论=是
```

### 0.5.3 严禁出现的内容（违者本轮失败）

| ❌ 禁止 | 替代做法 |
|---|---|
| `（沉稳起立）` `（拍桌）` `（深呼吸）` 等动作括号 | 删除，直接进入发言内容 |
| `（眉头紧锁）` `（语速加快）` 等表情/语气括号 | 删除 |
| `（打断）` `（转向评委）` 等舞台说明 | 删除；如需表示打断，靠对话顺序自然呈现 |
| `马薇薇站起，犀利地扫视全场` 等小说旁白 | 删除整句 |
| "现场气氛紧张" "全场鸦雀无声" 等环境描写 | 删除整句 |
| "作为正方一辩，我现在开始我的申论" 等元说明 | 改成 "谢谢主席，各位评委……" |

## 1. 隔离模式（Isolated · 默认）

### 1.1 阶段 0 · 启动（每位辩手一次，共 8 次，可并行）

```text
你正在执行一场角色辩论。你的唯一任务是以 {persona} 的风格作答。

Step 1  调用 Skill(skill="{persona_skill}")，严格遵守其角色扮演 DNA：
        - 用「我」第一人称；
        - 不得有元评论（"作为一名 AI..."），不得跳出角色；
        - 不得自报"如果是 {persona}，他会说..."（直接以他/她身份说）。

Step 2  记住以下场景配置：
        - 辩题: {topic}
        - 你的辩位: {position}
        - 你的持方: {side}
        - 队友: {三位同侧辩手}
        - 对手: {四位对侧辩手}
        - 全场基准语速: 280 字/分钟

Step 3  必须遵守的两个格式规范：
        (a) 真实辩稿格式 —— rules/script_format.md §1-§5
            · 严禁（动作/表情/语气）括号，严禁小说式旁白；
            · 必须用"谢谢主席""开宗明义""综上所述""第一/第二/第三"等真实辩场套语；
            · 内容必须是辩手要"逐字念出来"的话。
        (b) 现场对话呈现规范 —— §0.5
            · 申论/总结陈词的标题 + 正文格式；
            · 质询/对话的辩位加粗 + 冒号 + Q-A 交替格式；
            · 结尾附【完成】契约行。

确认收到，回复「{persona} 就位」即可，不要输出任何辩论内容。
```

### 1.2 每轮增量发送

```text
当前是 {round_name}。计时: {时长}（等价字数下限 ≥ {N} 字）。

【你需要先看的新信息】
{prev_transcript_delta}

【本轮你的任务】
{round_inst}  ← 完整从下方 §3 / §4 / §5 / §6 / §7 复制过来

【对话呈现要求】
1. 按 §0.5 的标题 + 正文格式输出；
2. 严守 rules/script_format.md 的辩稿格式（无动作括号、用辩场套语）；
3. 正文必须达到时长对应的字数下限（少于即重做）；
4. 结尾附完成契约行。

只返回完整对话呈现 + 完成契约。不要前言、不要"以下是我的发言"、不要摘要。
```

### 1.3 子代理坏掉了怎么办

| 症状 | 处理顺序 |
|---|---|
| 返回了元评论或前言 | ① 同一代理重试，"只返回正文 + 完成契约" |
| 字数低于时长下限 | ② 重发，"上次只有 X 字 (≈ N 秒)，本轮计时 {时长} 下限 Y 字" |
| 没有附【完成】契约行 | ③ 重发，"请补上完成契约行" |
| **出现（动作）/（情绪）等小说式标记** | ④ **重发，"违反 script_format.md §1.1，请删除所有括号、舞台说明、小说旁白"** |
| 用了"已落盘 X.md"代替发言 | ⑤ 重发，"违反对话呈现规范，必须在对话里完整演" |
| 仍失败 | ⑥ 重启代理 + 紧凑回顾 |
| 角色明显漂移 | ⑦ 强制重启 + 贴该辩手金句/口头禅 |

### 1.4 Token 节约（不能牺牲完整度）

- 启动一次，多次发消息；
- 增量上下文 = "其他辩手最近一轮的发言" + "本轮指令" + "完成契约要求"；
- **绝不**因为省 token 把时长/字数下限调低；
- 超过 5 轮或发现锚定，做一次受控重启。

## 3. 阶段 B · 赛前规划 Prompts（对话讨论 + 编排器提炼）

> 阶段 B 是整场辩论的地基。**v2.5 起新增 §3B.0 历史实录检索**——
> 在自己埋头立论之前，先去 debatetimer.cn 看看别人怎么打的。
> 然后才进入 §3B.1–§3B.5 立论讨论。讨论结束后**才**由编排器从对话
> 提炼成 `{side}-备赛资料.md`（按 `rules/script_format.md` §6.1）。

### §3B.0  历史实录检索（★ v2.5 新增 · 约 8 分钟 · 必做）

```text
辩题: {topic}
持方: {side}
四位辩手: {p1, p2, p3, p4}

任务：在 debatetimer.cn 上检索同题或近似辩题的过往辩论实录，
整理成一份「敌情通报」作为后续立论的参考。

详细方法论见 `knowledge/06-历史实录检索.md`（必读）。

【执行步骤】

**Step 1 · 关键词检索（≈1 分钟）**
打开 https://debatetimer.cn/record （用 fetch_page 工具）。
查阅主页列出的最近实录，**按以下优先级**尝试匹配本辩题：

1. 完整辩题搜索：「{topic}」
2. 核心词组合：「{核心词1} {核心词2}」（空格分隔即"且"）
3. 主题词：「{核心词1}」
4. 同类辩题：「{相关主题}」

注意：debatetimer 的搜索"可以理解成一个百度搜索"（官方说明）。

**Step 2 · 抓取实录文本（≈3 分钟）**
对每个相关 record URL（形如 https://debatetimer.cn/record/{uuid}），
用 fetch_page 抓取，提取以下要素：

- 比赛名 + 日期
- 正反方辩题原文
- 每位辩手的【论述流程】6-8 步清单（AI 整理段，每篇都有）
- 关键金句 / 例子 / 数据
- 评委判决（若实录中有）

如果命中 > 5 场，优先抓取「最近 3 场」+「评委明确判决的 2 场」。

**Step 3 · 四人讨论并整理敌情通报（≈4 分钟）**
四位辩手按 一辩→二辩→三辩→四辩 顺序，每人 ≥ 100 字汇报：

**{side}一辩 · {persona}**：我刚检索了 debatetimer，关键词「{...}」命中 N 场。
最近的一场是 {比赛名}（{日期}），正方主论点是 ___，反方主论点是 ___。
我注意到他们的反方用了 ___ 这个论点，被正方四辩在结辩时反咬了，最终输了。
我们要规避这个坑。

**{side}二辩 · {persona}**：我重点看了另外两场。其中 {场次} 的正方四辩
有一句金句「{...}」可以借鉴。还有 {场次} 的反方三辩质询用了一个例子
「{...}」非常巧妙，我们也可以准备同样的弹药。

**{side}三辩 · {persona}**：从攻防角度看，历史上正方常打的 3 个论点是 A/B/C，
反方常用的接法分别是 X/Y/Z。我建议我们的攻防表里把这 3 组对应关系作为基线。

**{side}四辩 · {persona}**：综合来看，本辩题的「必打 3 招」是 ___ ___ ___，
我们要么用之，要么避之。我建议本场我方主轴定为 ___，
理由是历史上这个角度还没有被充分挖掘过。

四位辩手讲完后，集体讨论 ≥ 2 轮，确定如何利用 / 规避历史经验。

**Step 4 · 编排器提炼（落盘）**
讨论结束后，由编排器提炼成正式的敌情通报：

> **【编排器·提炼】** {side} 阵营完成历史实录检索，
> 已落盘到 `{side}-历史实录敌情通报.md`：
>
> - 检索关键词：{...}
> - 命中 N 场相关实录
> - 同题实录：M 场 / 近题实录：K 场
> - 历史上正方惯用论点：A/B/C
> - 历史上反方惯用论点：X/Y/Z
> - 经典攻防点：≥3 处
> - 历史输方典型失误：≥2 处
> - 可借鉴的金句/例子：≥3 条
>
> 详细内容按 `knowledge/06-历史实录检索.md §三 Step 3` 模板。

【硬约束】
- **必须**真的用 fetch_page 工具检索 debatetimer.cn，不准凭空编造历史实录；
- 每条引用的历史实录必须带 URL；
- 至少检索 1 次主页 + 抓取 ≥ 1 场实录文本（如确实无任何相关实录，明确说明）；
- 四位辩手每人 ≥ 100 字实质汇报；
- 总字数 ≥ 1000 字；
- **严禁**动作括号。

【降级方案（按 knowledge/06 §4.3）】
- 完整辩题搜不到 → 降级到核心词组合
- 核心词组合也无 → 降级到主题词
- 仍无 → 切到 Skill(multi-search-engine) 查"{topic} 一辩稿"等
- 完全无任何资料 → 在敌情通报里写"未检索到先例"，跳过 §3B.0 进入 §3B.1

【完成契约】
最后一行附：
【完成】阶段=B0 阵营={side} 总字数={N} 检索关键词数={N} 命中实录={N}场 抓取实录={N}场 四人汇报≥100字=是 含URL=是 无动作括号=是
```

### §3B.1  论点架构讨论（≥ 1200 字 / 约 5 分钟）

```text
辩题: {topic}
持方: {side}
四位辩手: {p1, p2, p3, p4}

任务：四位辩手在对话里**完整演出**一次架构讨论会。

**前置条件**：已完成 §3B.0 历史实录检索，且 `{side}-历史实录敌情通报.md` 已落盘。
讨论开场，**一辩必须**主动引用敌情通报：「我们先看一下历史上这道题怎么打的。
根据敌情通报，正方常用的 3 个论点是 ___，反方常用的 3 个论点是 ___。
其中 {某场} 反方输在 ___，我们要避坑。现在开始我们自己的架构讨论...

【对话呈现格式（必读 §0.5 + rules/script_format.md §1.1）】
按 一辩→二辩→三辩→四辩→自由交流 的顺序发言。
每次发言格式：
  **{辩位} · {辩手}**：{发言内容}

注意：
- **不要**带（动作/表情/语气）括号；
- **不要**写"沉吟两秒""翻笔记""拍桌"等小说式描写；
- 直接写讨论内容，靠对话本身体现讨论氛围。

整场讨论必须**完整展开**以下 4 个议题：

议题 1 · 定义争夺（≥ 4 轮发言）
```

## 9. 失败重做触发条件汇总（速查）

| 触发条件 | 必做 |
|---|---|
| 字数低于该轮时长换算下限 | 重发 prompt，"上次只有 X 字 (≈ Y 秒)，本轮 Z 秒下限 W 字" |
| 没有【完成】契约行 | 重发 prompt，"请补上完成契约行" |
| 出现"已生成 X.md"代替对话 | 重发 prompt，"违反 §0.5 对话呈现规范" |
| **出现（动作/表情/语气）括号** | **重发 prompt，"违反 rules/script_format.md §1.1，请删除所有括号、舞台说明、小说旁白"** |
| **出现小说式旁白（"马薇薇站起"等）** | **重发 prompt，"违反 rules/script_format.md §1.1，请改成可直接朗读的辩稿"** |
| 质询回合数不足 | 重发 prompt，"上次 N 个回合，本轮下限 M 个" |
| 自证打岔超 3 次 | 重发 prompt，"打岔方违反规则，必须 ≤ 3 次" |
| 申论辩手抄了一辩已说内容 | 重发 prompt，"二辩/三辩不准重复一辩" |
| 总结陈词没回应对方刚才的总结 | 重发（特指 R14 没回应 R13） |
| 评委判平局/双方都好 | 重发 prompt，"必须二选一胜负方" |
| 赛后 7 份稿件合并 | 重发对应 §6.X，"必须 N 份独立文件 + N 次彩排" |
| **落盘文件含动作括号或占位符** | **重发对应 §6.X，"必须通过 rules/script_format.md §7.3 自检"** |
